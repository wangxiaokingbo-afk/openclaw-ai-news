# 广义 POI 提取模型 - 微调训练方案

## 🎯 目标定义

### 传统 POI vs 广义 POI

| 类型 | 传统 POI | 广义 POI（我们的目标） |
|------|---------|---------------------|
| **景点** | 故宫、西湖 | 最佳拍照点、日落观赏点 |
| **住宿** | 酒店、民宿 | 露营地点、青旅 |
| **餐饮** | 餐厅、咖啡馆 | 小吃摊、野餐点 |
| **交通** | 地铁站、公交站 | 徒步起点、停车点、补给点 |
| **购物** | 商场、超市 | 集市、特产店 |
| **新增** | - | 打卡点、机位、观景台、洗手间、充电桩 |

### 广义 POI 分类体系

```
POI
├── 导航类（可到达）
│   ├── 标准地点：景点/酒店/餐厅/车站
│   ├── 非标准地点：打卡点/拍照点/观景台
│   └── 功能点位：补给点/停车点/洗手间/充电桩
├── 搜索类（用户会搜）
│   ├── 热门关键词：最佳机位/日出观测点
│   └── 长尾关键词：免费拍照点/宠物友好
└── 决策类（帮助选择）
    ├── 推荐点位：值得去/必打卡
    └── 避雷点位：坑/不要去
```

## 📊 数据标注方案

### 标注数据类型

**1. 帖子数据**
- 标题
- 正文
- 标签（#话题）
- 点赞/评论/收藏数

**2. 评论数据**
- 评论内容
- 回复关系
- 点赞数

**3. POI 标注**
```json
{
  "text": "推荐去这个观景台看日落，在山顶的右侧",
  "pois": [
    {
      "name": "观景台",
      "type": "拍照点",
      "category": "导航类",
      "span": [5, 8],
      "attributes": {
        "function": "看日落",
        "location": "山顶右侧",
        "sentiment": "推荐"
      }
    }
  ]
}
```

### 标注类别体系

| 一级分类 | 二级分类 | 示例 |
|---------|---------|------|
| 导航类 - 标准 | 景点/酒店/餐厅/交通 | 故宫、丽思卡尔顿 |
| 导航类 - 非标准 | 拍照点/打卡点/观景台 | 最佳机位、日落观测点 |
| 导航类 - 功能 | 补给点/停车/洗手间 | 徒步补给站、充电桩 |
| 搜索类 - 热门 | 高频搜索词 | 免费拍照、宠物友好 |
| 搜索类 - 长尾 | 特定需求 | 人少景美、不用排队 |
| 决策类 - 推荐 | 正面推荐 | 值得去、必打卡 |
| 决策类 - 避雷 | 负面预警 | 坑、不要去 |

## 🔧 模型选择

### 方案对比

| 方案 | 模型 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|-------|
| **方案 A** | BERT + NER | 成熟、快速 | 需要标注数据 | ⭐⭐⭐⭐ |
| **方案 B** | RoBERTa + CRF | 准确率更高 | 训练复杂 | ⭐⭐⭐⭐ |
| **方案 C** | ChatGLM3/Qwen | 零样本能力强 | 推理慢、成本高 | ⭐⭐⭐ |
| **方案 D** | 规则 + 小模型 | 快速启动 | 准确率有限 | ⭐⭐⭐ |

### 推荐方案：BERT + 规则混合

**第一阶段（快速启动）**：规则 + 关键词匹配
- 2 周内上线
- 准确率 60-70%

**第二阶段（模型微调）**：BERT + NER
- 1-2 个月
- 准确率 80-85%

**第三阶段（持续优化）**：大模型辅助
- 3 个月 +
- 准确率 90%+
