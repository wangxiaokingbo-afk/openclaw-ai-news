# 广义 POI 数据标注规范 v1.0

## 📋 标注目标

训练一个能识别**广义 POI**的模型，包括：
1. 标准 POI（景点/酒店/餐厅）
2. 非标准 POI（打卡点/拍照点/观景台）
3. 功能点位（补给点/停车场/洗手间）

---

## 🏷️ 标注类别

### POI 分类体系

```
POI
├── 导航类
│   ├── 景点：景区/公园/海滩/山/湖
│   ├── 住宿：酒店/民宿/客栈
│   ├── 餐饮：餐厅/咖啡馆/小吃店
│   ├── 交通：机场/火车站/地铁站
│   ├── 拍照点：打卡点/机位/观景台/拍摄点
│   ├── 功能点：补给点/停车点/露营点/售票处
│   ├── 设施点：洗手间/充电桩/加油站/饮水处
│   └── 观测点：日出观测点/日落观赏点/观星点
├── 搜索类
│   ├── 热门：免费/人少/宠物友好
│   └── 长尾：不用排队/性价比高
└── 决策类
    ├── 推荐：值得去/必打卡
    └── 避雷：坑/不要去
```

## 🎯 标注规则

### 什么应该标注为 POI？

**✅ 是 POI**：
- 用户会搜索的地点
- 可以导航到达的点
- 有实际功能的点位

**示例**：
```
"推荐去这个观景台看日落" → 观景台 ✅
"旁边有补给点可以买水" → 补给点 ✅
"免费拍照点，人少景美" → 拍照点 ✅
"景区入口有停车场" → 停车场 ✅
```

**❌ 不是 POI**：
- 抽象概念（如"风景"、"氛围"）
- 动作描述（如"徒步"、"爬山"）
- 时间描述（如"日出"、"黄昏"）

**示例**：
```
"风景很美" → 风景 ❌（抽象概念）
"徒步 30 分钟" → 徒步 ❌（动作）
"看日出" → 日出 ❌（时间，但"日出观测点"✅）
```

## 📊 标注示例

### 示例 1：标准 POI

```json
{
  "text": "推荐去三亚的亚龙湾和蜈支洲岛",
  "pois": [
    {"name": "亚龙湾", "type": "景点", "category": "导航类"},
    {"name": "蜈支洲岛", "type": "景点", "category": "导航类"}
  ]
}
```

### 示例 2：非标准 POI

```json
{
  "text": "山顶有个观景台，是看日落的最佳机位",
  "pois": [
    {"name": "观景台", "type": "拍照点", "category": "导航类"},
    {"name": "最佳机位", "type": "拍照点", "category": "导航类"}
  ]
}
```

### 示例 3：功能点位

```json
{
  "text": "景区入口有停车场和充电桩，旁边还有洗手间",
  "pois": [
    {"name": "停车场", "type": "功能点", "category": "导航类"},
    {"name": "充电桩", "type": "设施点", "category": "导航类"},
    {"name": "洗手间", "type": "设施点", "category": "导航类"}
  ]
}
```

### 示例 4：评论中的 POI

```json
{
  "text": "请问观景台怎么去？旁边有补给点吗？",
  "pois": [
    {"name": "观景台", "type": "拍照点", "category": "导航类"},
    {"name": "补给点", "type": "功能点", "category": "导航类"}
  ],
  "user_intent": "导航"
}
```

## 🛠️ 标注工具

### 推荐工具

1. **doccano**（开源免费）
   - 支持序列标注
   - 支持文本分类
   - Web 界面友好

2. **Label Studio**（开源）
   - 功能强大
   - 支持多种标注类型
   - 可协作

3. **自研工具**（基于 Streamlit）
   - 定制化强
   - 集成工作流

### 安装 doccano

```bash
pip install doccano
doccano createuser --username admin --password admin
doccano webserver --port 8000
```

## 📈 质量控制

### 标注一致性

- **多人标注**: 每条数据至少 2 人标注
- **一致性检验**: Kappa 系数 > 0.7
- **争议处理**: 第三人仲裁

### 验收标准

- **精确率**: > 85%
- **召回率**: > 80%
- **F1 分数**: > 82%

1. **数据收集** → 爬取小红书笔记
2. **数据清洗** → 去重、过滤

4. **人工校验** → 修正预标注结果
5. **质量检查** → 一致性检验
6. **导出训练** → JSONL 格式
